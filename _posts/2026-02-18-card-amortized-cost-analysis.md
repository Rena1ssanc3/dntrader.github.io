## 鉴定机制

每次鉴定消耗固定素材，随机产出一个等级的卡片：

| 等级 | 概率 |
|------|------|
| C    | 34%  |
| B    | 40%  |
| A    | 20%  |
| S    | 5%   |
| L    | 1%   |

每次鉴定都会产出一张卡片，不存在"失败"——所有结果都有价值。

## 摊还分析逻辑

### 问题

朴素计算（几何分布期望）会高估单张卡片的成本。例如 L 卡概率 1%，朴素期望需要 100 次鉴定才能出一张，但这 100 次中产出的其他 99 张卡片也有价值。摊还分析将总成本公平分摊到所有产出的卡片上。

### 权重分配

按稀有度分配权重——越稀有的卡片承担越多成本。自然权重为概率的倒数（平均多少次出一张）：

| 等级 | p    | w = 1/p |
|------|------|---------|
| C    | 0.34 | 2.94    |
| B    | 0.40 | 2.50    |
| A    | 0.20 | 5.00    |
| S    | 0.05 | 20.00   |
| L    | 0.01 | 100.00  |

### 推导

N 次鉴定，总素材消耗：

```
total_cost = N × cost[m]
```

加权产出总量：

```
total_weight = Σ N × p[g] × w[g] = Σ N × p[g] × (1/p[g]) = 5N
```

每单位权重的成本：

```
cost_per_weight = N × cost[m] / 5N = cost[m] / 5
```

某等级卡片的摊还成本：

```
amortized_cost(grade, material) = cost[m] / (5 × p[grade])
```

### 摊还系数表

以单次鉴定消耗 1 单位素材为基准：

| 等级 | 摊还系数 |
|------|---------|
| C    | 0.588   |
| B    | 0.500   |
| A    | 1.000   |
| S    | 4.000   |
| L    | 20.000  |

> B 卡的摊还成本最低（概率最高），L 卡是 B 卡的 40 倍。

### 验证

以副官翁扎卡（鉴定消耗：玛瑙 2、阿尔泰 2）为例，100 次鉴定消耗 200 阿尔泰：

- 34 张 C × 1.18 = 40.0
- 40 张 B × 1.00 = 40.0
- 20 张 A × 2.00 = 40.0
- 5 张 S × 8.00 = 40.0
- 1 张 L × 40.00 = 40.0
- **合计 = 200.0** ✓
